# 다른 기기에서 접속 안 되는 문제 해결

## 🔴 문제: 노트북에서는 되는데 데스크탑에서는 안 됨

## 🔍 가능한 원인 및 해결 방법

### 1. 브라우저 캐시 문제

**증상:** 사이트가 열리지 않거나 오래된 버전이 표시됨

**해결 방법:**

1. **브라우저 캐시 완전 삭제**
   - Chrome: 설정 → 개인정보 및 보안 → 인터넷 사용 기록 삭제
   - "캐시된 이미지 및 파일" 선택
   - "전체 기간" 선택
   - "데이터 삭제" 클릭

2. **시크릿 모드로 테스트**
   - Ctrl+Shift+N (Windows) 또는 Cmd+Shift+N (Mac)
   - 시크릿 모드에서 접속 테스트

3. **하드 리프레시**
   - Ctrl+Shift+R (Windows) 또는 Cmd+Shift+R (Mac)
   - 또는 Ctrl+F5

### 2. DNS 문제

**증상:** 사이트에 접속할 수 없음 (DNS_PROBE_FINISHED_NXDOMAIN 등)

**해결 방법:**

1. **다른 DNS 서버 사용**
   - Google DNS: `8.8.8.8`, `8.8.4.4`
   - Cloudflare DNS: `1.1.1.1`, `1.0.0.1`

2. **DNS 캐시 삭제**
   - Windows: `ipconfig /flushdns` (관리자 권한)
   - Mac: `sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder`

3. **다른 브라우저로 테스트**
   - Chrome, Firefox, Edge 등 다른 브라우저로 접속

### 3. 네트워크/방화벽 문제

**증상:** 특정 사이트만 접속 안 됨

**해결 방법:**

1. **방화벽 확인**
   - Windows 방화벽 설정 확인
   - 백신 소프트웨어 방화벽 확인
   - 회사/학교 네트워크 정책 확인

2. **프록시 설정 확인**
   - 브라우저 설정 → 네트워크 → 프록시 설정
   - 불필요한 프록시 비활성화

3. **VPN 연결 확인**
   - VPN이 켜져 있으면 끄고 테스트

### 4. CORS 설정 미적용

**증상:** Google 로그인은 되는데 API 호출 실패, CORS 오류

**해결 방법:**

1. **Render 배포 완료 확인**
   - Render 대시보드에서 서비스 상태가 "Live"인지 확인
   - 최근 배포가 성공했는지 확인

2. **환경 변수 확인**
   - Render → Environment 탭
   - `ALLOWED_ORIGINS`에 GitHub Pages URL이 포함되어 있는지 확인

3. **서비스 재시작**
   - Render에서 "Manual Deploy" 클릭하여 재시작

### 5. 브라우저 확장 프로그램 문제

**증상:** 특정 기능만 작동 안 함

**해결 방법:**

1. **확장 프로그램 비활성화**
   - 시크릿 모드로 테스트 (확장 프로그램 자동 비활성화)
   - 또는 확장 프로그램 일시 비활성화

2. **광고 차단기 확인**
   - 광고 차단기가 API 요청을 차단할 수 있음
   - 일시 비활성화 후 테스트

### 6. 시간 동기화 문제

**증상:** 인증 오류, 토큰 만료 오류

**해결 방법:**

1. **시스템 시간 확인**
   - 데스크탑의 시스템 시간이 정확한지 확인
   - 자동 시간 동기화 활성화

### 7. HTTPS 인증서 문제

**증상:** "연결이 비공개로 설정되어 있지 않습니다" 오류

**해결 방법:**

1. **인증서 확인**
   - 브라우저 주소창의 자물쇠 아이콘 확인
   - 인증서 정보 확인

2. **시스템 날짜 확인**
   - 시스템 날짜가 잘못되어 있으면 인증서 오류 발생 가능

## 🐛 구체적인 오류별 해결

### "사이트에 연결할 수 없음"

**원인:** DNS 문제, 네트워크 문제

**해결:**
1. DNS 서버 변경
2. 네트워크 재연결
3. 다른 네트워크로 테스트

### "CORS 오류"

**원인:** 백엔드 CORS 설정 미적용

**해결:**
1. Render 대시보드에서 `ALLOWED_ORIGINS` 확인
2. GitHub Pages URL 포함 확인
3. 서비스 재시작

### "Google 로그인 실패"

**원인:** Google OAuth 설정 문제

**해결:**
1. Google Cloud Console에서 설정 확인
2. "승인된 JavaScript 원본" 확인
3. "승인된 리디렉션 URI" 확인

### "API 연결 실패"

**원인:** 백엔드 서버 문제, CORS 문제

**해결:**
1. Render 대시보드에서 서비스 상태 확인
2. 백엔드 로그 확인
3. CORS 설정 확인
