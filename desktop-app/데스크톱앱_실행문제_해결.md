# 데스크톱 앱 실행 문제 해결

## 🔴 문제: 앱이 실행되지 않음

## 🔍 가능한 원인

### 1. 백엔드 서버 미실행
데스크톱 앱은 백엔드 API 서버가 필요합니다.

**해결 방법:**
1. 백엔드 서버 실행 필요
2. 또는 백엔드를 앱에 포함

### 2. API URL 설정 문제
앱이 백엔드 서버를 찾지 못함

**현재 설정:**
- 개발 모드: `http://localhost:5173` (프론트엔드)
- 프로덕션: 빌드된 파일 사용
- API URL: 환경 변수 또는 기본값 `http://localhost:8001`

### 3. macOS 보안 설정
확인되지 않은 개발자 경고

**해결 방법:**
1. 시스템 설정 → 보안 및 개인 정보 보호
2. "확인되지 않은 개발자" 허용
3. 또는 앱을 우클릭 → 열기

## ✅ 해결 방법

### 방법 1: 백엔드 서버 실행 (필수)

**터미널에서:**
```bash
cd backend
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8001
```

백엔드 서버가 실행 중이어야 앱이 작동합니다.

### 방법 2: 앱 실행 확인

1. **Applications에서 앱 실행**
   - Applications 폴더에서 "학년 배정 시스템" 더블클릭
   - 또는 Spotlight에서 검색

2. **보안 경고 해결**
   - "확인되지 않은 개발자" 경고 시
   - 시스템 설정 → 보안 → "그래도 열기" 클릭

3. **콘솔 로그 확인**
   - 앱 실행 후 개발자 도구 열기 (Cmd+Option+I)
   - 콘솔에서 오류 확인

### 방법 3: 완전한 오프라인 지원 (향후)

백엔드를 앱에 포함시키면 서버 없이도 작동합니다.

## 🚀 빠른 확인

1. **백엔드 서버 실행 확인**
   ```bash
   curl http://localhost:8001/docs
   ```
   응답이 오면 서버가 실행 중입니다.

2. **앱 실행**
   - Applications에서 앱 실행
   - 로그인 페이지가 보이면 정상

3. **오류 확인**
   - 개발자 도구 (Cmd+Option+I)
   - 콘솔 탭에서 오류 메시지 확인

## 📝 현재 상태

- ✅ DMG 파일: 생성 완료
- ⚠️ 백엔드 서버: 실행 필요
- ⚠️ API 연결: `http://localhost:8001` 필요

## 💡 완전한 해결책

**옵션 1: 백엔드 서버 항상 실행**
- 서버를 시작 프로그램에 추가
- 또는 서비스로 등록

**옵션 2: 백엔드를 앱에 포함 (권장)**
- Python 백엔드를 Electron 앱에 포함
- 앱 실행 시 자동으로 백엔드 서버 시작
- 완전히 독립적으로 작동

**옵션 3: SQLite 직접 사용**
- 백엔드 없이 프론트엔드에서 직접 DB 접근
- 가장 가벼운 방법
